# 导出面板本次云发布临时选项优化

## 背景

当前导出面板已支持“本次云发布”快捷设置，但只有“预签名 URL”和“可撤销链接”两个发布模式。该设置是一次性设置，只影响当前笔记/当前导出，不写回插件设置。

本次需要把它扩展为更完整的临时发布策略：

- 可在本次导出中关闭云发布，即使全局云发布配置已开启。
- 选择云发布时，可决定云端上传成功后是否保留本地 HTML 导出产物。
- 云发布关闭时不显示本地保留子选项，因为关闭云发布时用户一定需要本地 HTML 文件。

## 目标

- 导出面板“本次云发布”提供三个选项：关闭、预签名 URL、可撤销链接。
- “关闭”只影响本次导出，不修改全局 `Settings.cloudPublish.enabled`。
- “预签名 URL”和“可撤销链接”只在本次导出中临时启用云发布，不修改全局设置。
- 在云发布开启时显示子选项“云发布后保留本地 HTML 文件”。
- 子选项默认开启，保持本地产物，避免误删。
- 用户取消该子选项时，只在云发布成功后删除本次导出的整个本地产物。

## 非目标

- 不在插件设置页新增全局“上传后删除本地文件”配置。
- 不实现 Worker + KV 可撤销链接。
- 不删除或撤销云端 R2 object。
- 不改变 R2、WebDAV、上传策略等全局配置项。

## 交互设计

导出面板中“本次云发布”作为父选项：

- `关闭`
- `预签名 URL`
- `可撤销链接`

默认值按全局配置推导：

- 全局 `Settings.cloudPublish.enabled = false` 时，默认显示 `关闭`。
- 全局开启时，默认显示全局 `publishMode`。

当选择 `预签名 URL` 或 `可撤销链接` 时，显示子选项：

- 标题：`云发布后保留本地 HTML 文件`
- 描述：`关闭后，仅在本次云发布成功后删除本次导出的本地产物。`
- 默认值：开启。

当选择 `关闭` 时隐藏该子选项，并强制本次导出保留本地文件。

## 数据流

导出面板提交时构造临时 `cloudPublishSettings`：

- `关闭`：
  - `enabled = false`
  - `keepLocalFilesAfterPublish = true`
- `预签名 URL`：
  - `enabled = true`
  - `publishMode = "presigned-url"`
  - `createPresignedUrl = true`
  - `keepLocalFilesAfterPublish` 来自子选项
- `可撤销链接`：
  - `enabled = true`
  - `publishMode = "revocable-link"`
  - `createPresignedUrl = false`
  - `keepLocalFilesAfterPublish` 来自子选项

这些值只传给本次 `HTMLExporter.exportWithResult()`，不调用 `SettingsPage.saveSettings()`。

## 本地产物清理规则

删除动作只在本次云发布满足以下条件后执行：

- 用户取消了“云发布后保留本地 HTML 文件”。
- `uploadedCount > 0`。
- `failedCount === 0`。
- 发布模式是 `presigned-url`。
- 已成功生成 `presignedUrl`。

当前 `revocable-link` 仍为预留模式，因此不会自动删除本地产物，避免本地文件被删除但用户没有可用访问链接。

删除范围：

- 单文件导出：删除本次成功上传的单个 HTML 文件。
- 目录导出：删除本次成功上传的所有本地文件，并清理变空的目录。

实现上不直接递归删除用户选择的父目录，而是删除云发布实际成功上传的本地文件，再清理空目录。这样既满足“本次导出的整个本地产物”需求，也避免导出目录中存在用户手动放置文件时被误删。

## 错误处理

- 上传失败时不删除本地产物。
- 预签名 URL 生成失败时不删除本地产物。
- 本地删除失败时不影响导出和发布结果，只写入导出警告。
- 删除后仍在导出结果区显示云端发布链接。

## 验证计划

- 执行 `npm run build`，确保 TypeScript 类型和打包通过。
- 执行 `git diff --check`，检查空白错误。
- 人工检查导出面板逻辑：
  - 云发布关闭时不显示保留子选项。
  - 选择预签名 URL 后显示保留子选项。
  - 选择关闭后本次 `enabled = false`。
  - 不保留本地文件时，只有成功发布并生成链接后才触发清理。
